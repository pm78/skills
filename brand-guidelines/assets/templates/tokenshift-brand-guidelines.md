## 2. Logo

### 2.1 Primary Logo — Horizontal Lockup

The primary logo is a horizontal lockup consisting of:
- **Mark**: Two interlocking ring outlines — one in midnight navy, one in cyan — tilted at dynamic angles representing integration and transformation.
- **Wordmark**: TOKENSHIFT in clean geometric sans-serif (Inter Bold). TOKEN in navy, SHIFT in cyan.
- **Tagline** (optional): "From code to culture." in slate, below the wordmark.

### 2.2 Logo Versions

| Version | Use Case |
|---------|----------|
| **Horizontal + tagline** | Primary use. Presentations, website header, letterhead. |
| **Horizontal (no tagline)** | Compact spaces. Email signatures, social media. |
| **Icon only** | Favicon, app icon, social avatar, small placements. |
| **Reversed (white)** | On dark or photographic backgrounds. |

### 2.3 Clear Space

Minimum clear space around the logo = **height of one ring** on all sides.
No text, images, or other elements may encroach on this space.

### 2.4 Minimum Size

- Horizontal lockup: minimum width **120px** (digital) / **30mm** (print).
- Icon only: minimum **24px** (digital) / **8mm** (print).

### 2.5 Logo Don'ts

- Do not rotate the logo.
- Do not stretch or distort.
- Do not apply gradients, shadows, or effects.
- Do not change the ring colors.
- Do not rearrange the mark and wordmark.
- Do not place on busy or low-contrast backgrounds.
- Do not outline the logo.
- Do not animate the rings separately.

## 4. Typography

### 4.1 Primary Typeface — Inter

**Inter** is the sole brand typeface. It provides a clean, geometric, highly legible system across all media.

| Element | Weight | Size (web) | Size (print) | Size (slides) |
|---------|--------|-----------|-------------|---------------|
| **H1 / Hero** | Bold (700) | 48px | 36pt | 44pt |
| **H2 / Section** | Bold (700) | 36px | 28pt | 36pt |
| **H3 / Subsection** | Semi-Bold (600) | 24px | 20pt | 24pt |
| **H4 / Card title** | Semi-Bold (600) | 20px | 16pt | 20pt |
| **Body** | Regular (400) | 16px | 11pt | 16pt |
| **Body small** | Regular (400) | 14px | 9pt | 14pt |
| **Caption / Label** | Medium (500) | 12px | 8pt | 12pt |
| **Overline** | Semi-Bold (600) | 12px uppercase | 8pt uppercase | 11pt uppercase |

### 4.2 Fallback Stack

```
font-family: 'Inter', -apple-system, BlinkMacSystemFont, 'Segoe UI', 'Helvetica Neue', Arial, sans-serif;
```

### 4.3 Monospace (code/data)

```
font-family: 'JetBrains Mono', 'Fira Code', 'Consolas', monospace;
```

### 4.4 Line Height & Spacing

- **Headings**: line-height 1.2
- **Body**: line-height 1.6
- **Paragraph spacing**: 1em between paragraphs.
- **Letter spacing**: 0 for body, +0.02em for overlines/labels.

### 4.5 Print Fallback

When Inter is unavailable (internal documents, legacy systems):
- **Heading**: Helvetica Neue Bold / Arial Bold
- **Body**: Calibri / Helvetica Neue Regular

## 5. Tone of Voice

### 5.1 Core Principles

1. **Direct** — Say it plainly. No jargon walls. No consultant-speak.
2. **Expert** — We know this space. We've implemented it. We speak from experience.
3. **Outcome-focused** — Lead with results and impact, not features or process.
4. **Transformative** — We move things forward. Every sentence should feel like progress.

### 5.2 Writing Style

- **Active voice** over passive. "We redesign workflows" not "Workflows are redesigned."
- **Short sentences**. If it needs a semicolon, split it.
- **Concrete specifics** over vague claims. "4-phase implementation" not "comprehensive approach."
- **Verbs over nouns**. "Implement" not "Implementation facilitation."

### 5.3 Words We Use

| Use | Instead of |
|-----|------------|
| Implement | Deploy, roll out |
| Transform | Disrupt, revolutionize |
| Shift | Change, pivot |
| Build | Develop, create |
| Embed | Integrate, incorporate |
| Measurable | Tangible, concrete |
| Workflow | Process, procedure |
| Team | Resource, FTE |

### 5.4 Words We Avoid

- Compliance, audit, risk mitigation (as primary framing)
- Synergy, leverage (as verbs), paradigm
- Best-in-class, world-class, cutting-edge
- Solution (as a standalone noun)
- Journey (without a destination)

## 6. Imagery & Illustration

### 6.1 Photography Style

- **Setting**: Modern offices, collaborative spaces, boardrooms with natural light.
- **Subjects**: Diverse teams working together. AI dashboards visible but not dominant.
- **Mood**: Confident, forward-looking, warm but professional.
- **Lighting**: Clean, natural, soft shadows.

### 6.2 Imagery Don'ts

- No shields, locks, or security imagery.
- No robot vs. human conflict metaphors.
- No generic stock handshakes.
- No neon/cyberpunk aesthetics.
- No fear-based or alarmist visuals.

### 6.3 Icons

- Style: Line icons, 1.5px stroke, rounded caps.
- Color: Navy or cyan on white; white on navy.
- Source: Lucide, Phosphor, or Heroicons (outline variant).

### 6.4 Data Visualization

- Primary data color: Cyan `#00B4D8`
- Secondary: Navy `#0F1B2D`
- Tertiary: Slate `#475569`
- Positive: Success green `#10B981`
- Negative: Error red `#EF4444`
- Grid/axis: Border gray `#CBD5E1`

## 7. Layout Principles

### 7.1 Grid

- **Web**: 12-column grid, max-width 1280px, 24px gutters.
- **Slides**: 16:9 ratio, 0.5" margins, content within safe zone.
- **Print**: A4, 20mm margins, baseline grid at 12pt.

### 7.2 Spacing Scale

Based on 4px base unit:

| Token | Value | Use |
|-------|-------|-----|
| `xs` | 4px | Tight inline spacing |
| `sm` | 8px | Input padding, icon gaps |
| `md` | 16px | Card padding, element gaps |
| `lg` | 24px | Section padding |
| `xl` | 32px | Major section gaps |
| `2xl` | 48px | Page section separation |
| `3xl` | 64px | Hero/footer breathing room |

### 7.3 Border Radius

| Element | Radius |
|---------|--------|
| Cards, panels | 8px |
| Buttons | 6px |
| Inputs | 4px |
| Avatars, badges | Full (50%) |
| Tags, chips | 4px |

### 7.4 Shadows

Subtle elevation only:
- **sm**: `0 1px 2px rgba(15, 27, 45, 0.05)`
- **md**: `0 4px 6px rgba(15, 27, 45, 0.07)`
- **lg**: `0 10px 15px rgba(15, 27, 45, 0.10)`
