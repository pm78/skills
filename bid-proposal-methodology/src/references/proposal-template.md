# Template de Proposition — Sections Obligatoires

## Instructions

Ce gabarit définit les sections obligatoires d'une proposition commerciale standard.
Adapter le contenu à chaque bid. Respecter l'ordre des sections sauf contrainte client contraire.
Les sections marquées `[CONDITIONNEL]` sont incluses uniquement si pertinentes.

---

## Structure type

### 0. Page de garde

| Élément | Contenu |
|---------|---------|
| Logo entreprise | En haut à gauche |
| Titre | Proposition — `<Intitulé du projet>` |
| Client | Nom du client |
| Référence | `BID-YYYY-NNN` |
| Date | Date de soumission |
| Mention confidentialité | « Document confidentiel — usage restreint au destinataire » |
| Version | `v1.0` |

### 1. Sommaire exécutif (1-2 pages max)

**But** : Convaincre un décideur pressé en 2 minutes.

Contenu obligatoire :
- Reformulation du besoin client en 2-3 phrases
- Proposition de valeur : ce que nous apportons de différenciant
- 3-5 win themes (messages clés)
- Résultat attendu : bénéfice mesurable pour le client
- Montant global et durée

**Règles** :
- Pas de jargon technique
- Chaque phrase doit apporter de la valeur au lecteur
- Orienté bénéfices client, pas features fournisseur

### 2. Compréhension du contexte et du besoin (2-3 pages)

Contenu obligatoire :
- Contexte stratégique du client (marché, enjeux, transformation)
- Reformulation détaillée du besoin
- Enjeux et objectifs identifiés
- Contraintes et prérequis compris
- Périmètre confirmé (in-scope / out-of-scope)

**Règle** : Démontrer qu'on comprend le besoin mieux que le client ne l'a formulé.

### 3. Solution proposée (3-5 pages)

Contenu obligatoire :
- Vision de la solution
- Architecture fonctionnelle / organisationnelle
- Composants clés de la solution
- Choix technologiques ou méthodologiques (avec justification)
- Valeur ajoutée par rapport aux alternatives
- Hypothèses de travail

**Règle** : Répondre directement aux critères d'évaluation techniques.

### 4. Méthodologie et démarche (2-3 pages)

Contenu obligatoire :
- Phases du projet avec objectifs de chaque phase
- Activités clés par phase
- Livrables par phase
- Points de contrôle / jalons
- Gouvernance du projet (comités, fréquence, participants)
- Approche qualité

Pour un bid TokenShift, utiliser les 4 phases standard :
1. **Diagnose** : AI Readiness Assessment
2. **Build** : Agentic Process Redesign, LLM/RAG Deployment
3. **Transition** : Workforce Upskilling, Change Management
4. **Assure** : Ongoing Governance, Optimization

### 5. Planning et jalons (1-2 pages)

Contenu obligatoire :
- Macro-planning (Gantt ou tableau)
- Jalons clés avec dates
- Dépendances client identifiées
- Critères de passage de phase

Format recommandé :

| Phase | Durée | Début | Fin | Jalons | Livrables |
|-------|-------|-------|-----|--------|-----------|
| | | | | | |

### 6. Équipe projet (1-2 pages)

Contenu obligatoire :
- Organigramme projet
- Profils clés (rôle, séniorité, compétences pertinentes)
- CV synthétiques en annexe (si demandés)
- Taux de couverture (% du temps par profil)

**Règle** : Mettre en avant l'adéquation profil ↔ besoin, pas le CV complet.

### 7. Références et expérience (1-2 pages)

Contenu obligatoire :
- 3-5 références pertinentes (secteur, taille, nature similaires)
- Pour chaque référence : client (anonymisé si nécessaire), contexte, solution, résultat mesurable
- Certifications et accréditations pertinentes

Format recommandé par référence :

| Élément | Contenu |
|---------|---------|
| Client | `<Secteur> — <Taille>` |
| Contexte | 2-3 lignes |
| Solution | 2-3 lignes |
| Résultat | Chiffre clé / bénéfice mesurable |
| Durée | |
| Technologies | |

### 8. Offre financière (1-3 pages)

Contenu obligatoire :
- Synthèse du prix global
- Décomposition par phase / lot / poste
- Conditions de facturation et de paiement
- Hypothèses de volumétrie
- Ce qui est inclus / exclu du prix
- Conditions de révision (si mission longue)

Format recommandé :

| Phase | Profils | Jours | Prix HT (EUR) |
|-------|---------|-------|----------------|
| | | | |
| **Total** | | | |

**Règles** :
- Ne jamais afficher les TJM individuels sauf si demandé explicitement par le client
- Toujours présenter un prix forfaitaire ou par phase en premier
- Option : scénario haut/bas si pertinent

### 9. Gestion des risques `[CONDITIONNEL]` (1 page)

Contenu obligatoire si inclus :
- Top 5 risques identifiés
- Pour chaque : description, impact, probabilité, mitigation proposée
- Partage des responsabilités client/fournisseur

### 10. Conditions générales `[CONDITIONNEL]` (1 page)

Contenu obligatoire si inclus :
- Durée de validité de l'offre (standard : 90 jours)
- Conditions de propriété intellectuelle
- Confidentialité
- Responsabilité et limites
- Résiliation
- Renvoi aux CGV/CGP si applicable

### 11. Annexes `[CONDITIONNEL]`

Contenu potentiel :
- CV détaillés de l'équipe
- Détail technique de la solution
- Tableau de conformité (critère → section)
- Glossaire
- Planning détaillé
- Matrice RACI
